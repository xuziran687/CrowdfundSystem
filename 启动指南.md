# 项目启动指南（Hardhat + Vue/Vite）

本项目包含两部分：

- `contract/`：Hardhat 合约工程（本地链、部署脚本）
- `frontend/`：Vue 3 + Vite 前端（连接 MetaMask 与合约交互）

## 环境要求

- Node.js（建议 18+ 或 20+）
- npm（或兼容的包管理器）
- 浏览器安装 MetaMask

## 1) 安装依赖

在项目根目录分别安装合约与前端依赖：

```bash
cd contract
npm install
```

```bash
cd ../frontend
npm install
```

## 2) 启动本地 Hardhat 节点

打开一个终端：

```bash
cd contract
npx hardhat node
```

启动成功后会监听本地 RPC（通常为 `http://127.0.0.1:8545`），并在终端输出若干测试账号与私钥。

## 3) 部署合约并同步前端地址

再打开一个终端（保持上面的 `hardhat node` 不要停）：

```bash
cd contract
npx hardhat run scripts/deploy.js --network localhost
```

部署脚本会做两件事：

- 生成/更新 `contract/contract-addresses.json`
- 同步写入前端地址文件 `frontend/src/sdk/addresses.json`

如果你想确认地址是否已更新，可以查看：

- `contract/contract-addresses.json`
- `frontend/src/sdk/addresses.json`

## 4) 配置 MetaMask（连接本地链）

在 MetaMask 中添加/切换到本地网络（Hardhat 默认）：

- **RPC URL**：`http://127.0.0.1:8545`
- **Chain ID**：`31337`（十六进制是 `0x7a69`）
- **货币符号**：`ETH`

然后从 `npx hardhat node` 的输出里复制一个测试账号私钥，导入到 MetaMask，确保有测试 ETH 可用于交易。

## 5) 启动前端

打开新终端：

```bash
cd frontend
npm run dev
```

按终端提示打开 Vite 提供的本地地址（一般是 `http://localhost:5173`）。

进入页面后点击“连接钱包”，按提示切到 Hardhat 本地链即可使用：

- Staking Vault 面板：查看余额/质押/领取等
- Crowdfunding 面板：创建 Campaign、认购、Finalize、Claim/Refund 等

## 补充：链上 `public` 变量与前端读取

Solidity 里标记为 `public` 的状态变量，编译器会**自动生成**同名的只读 getter（与在 MetaMask / viem 里调 `eth_call` 读合约一致）。前端 [`frontend/src/sdk/index.js`](frontend/src/sdk/index.js) 里的 `readContract({ functionName: 'totalETH' })` 等，本质上就是在调这些 getter，**不需要**在合约里再手写一层仅透传存储的 `getTotalETH()` 之类函数。  
合成视图（例如金库里随时间增长的 `totalR()`、`userR(address)`）仍必须是显式 `view` 函数，不能用一个 `public uint256` 替代。

## 常见问题排查

### 1) 页面显示合约地址不对 / 调用失败

- 确认你**先启动节点**再执行部署：
  - `npx hardhat node`
  - `npx hardhat run scripts/deploy.js --network localhost`
- 确认 `frontend/src/sdk/addresses.json` 已被部署脚本更新。

### 2) MetaMask 连接不上本地 RPC

- 确认 `hardhat node` 正在运行且端口是 `8545`
- 确认你用的是 `http://127.0.0.1:8545`（不要写错成别的端口/协议）

### 3) 交易一直转圈 / nonce 错误

- 关闭页面重新连接钱包
- 确认 MetaMask 当前网络就是 Hardhat（ChainId 31337 / `0x7a69`）
- 必要时重启 `hardhat node` 并重新部署（注意重启后链数据会重置）
